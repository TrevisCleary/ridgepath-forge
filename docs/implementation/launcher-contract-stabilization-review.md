## Contracts Established

- `launcher-settings.json` is now the launcher-side contract configuration.
- Operations Library root, required folders, required files, templates, workflows, and prompt paths are explicitly configured.
- Project Management directory, dashboard file name, open-file allowlist, and supported dashboard schema versions are configurable.
- `docs/launcher-workflow-contract.md` documents workflow names, prompt paths, expected inputs, and expected outputs.
- `docs/project-management-contract.md` documents the Project Management file and schema contract.

## Remaining Hardcoded Assumptions

- `PROJECTS_ROOT` still defaults to `C:\Development\Projects` when not provided through environment variables.
- `OPERATIONS_LIBRARY_ROOT` still has a built-in fallback used when settings and environment variables are unavailable.
- `LAUNCHER_API_PORT`, `LAUNCHER_CLIENT_PORT`, `LAUNCHER_REDIRECT_PORT`, and local hostnames still have script/config defaults.
- Starter runtime files `package.json`, `server.js`, and project `README.md` remain generated by launcher code because they are local project-creation assets rather than Operations Library handoff artifacts.
- Vite proxy target and allowed hosts remain in `vite.config.js`.
- Host registration script still writes `dev-launcher` and `devlauncher`.
